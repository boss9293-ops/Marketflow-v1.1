## Discovery Summary
- Existing V3/V6 and DeepSeek implementations are script-driven generators that may call live APIs and write cache artifacts.
- Read-only cache access paths already exist and are safe candidates for MCP `existing_engine_safe` mode.
- Generate endpoints and script entry points are unsafe for default MCP runtime wiring due to side effects.
- Safe MCP default should remain offline placeholder until explicit opt-in and stronger guardrails are approved.

# CODEX_MACRO_FRED4_FIRST

## 0) 스코프 고정 (이번 스텝)

Only FRED 4 series:

- WALCL (weekly)
- RRPONTSYD (daily)
- EFFR (daily)
- VIXCLS (daily)

Disable/Hide:

- BTC/GOLD/QQQ/XAP (not implemented in this step)

Output must still match the unified contract:

- LPI / RPI / VRI / MPS + confidence + drivers + updated/stale

## 1) Data Pipeline (FRED4)

### 1.1 Fetch & Cache

- Backend cron: daily refresh 1회 (EOD)

Store raw series as:

- `series_id, date, value, source="FRED", fetched_at`

WALCL weekly → daily alignment:

- forward-fill to daily dates for computations and UI display
- label `[Weekly]` in UI

### 1.2 Stale Policy (must follow policy)

- WALCL stale if last_update > 10 days
- RRP/EFFR/VIX stale if last_update > 2 trading days

stale triggers:

- stale badge
- confidence penalty
- never “zero out” score

## 2) Feature Engineering (raw → transformed)

### 2.1 Transform definitions (from `macro_policy_v1.json`)

#### WALCL_8W_CHANGE_PCT

- pct_change over 8 weekly points (or 8w equivalent)
- then direction = -1

#### RRP_20D_CHANGE_PCT

- pct_change over 20 daily points
- direction = -1

#### EFFR_LEVEL

- level, direction = +1

#### EFFR_1M_CHANGE_BP

- bp_change over 21 daily points (approx 1M)
- direction = +1

#### VIX_LEVEL

- level, direction = +1

#### VIX_5D_CHANGE_PCT

- pct_change over 5 daily points
- direction = +1

### 2.2 Winsorize

- winsorize each transformed feature before percentile:
  - low=1%, high=99% (per rolling window distribution)

### 2.3 Rolling percentile (rank-based)

Lookback points:

- daily: max 1260 points (~5y)
- weekly: max 260 points (~5y)

Percentile is empirical rank:

- `pct = 100 * (rank(x) - 1) / (N - 1)`

If `N < min_samples`:

- pct can still be computed if possible,
- but mark `data_limited=true` and apply confidence penalty

## 3) Index Computation (percentile weighted sum)

### 3.1 LPI (0-100)

- `LPI = 0.55PCT(-ΔWALCL_8w) + 0.45PCT(-ΔRRP_20d)`
- State bins: Easy / Neutral / Tight by 33/66

### 3.2 RPI (0-100)

- `RPI = 0.6PCT(EFFR_level) + 0.4PCT(ΔEFFR_1m_bp)`
- State bins: Easing / Stable / Restrictive by 33/66

### 3.3 VRI (0-100)

- `VRI = 0.6PCT(VIX_level) + 0.4PCT(VIX_5d_change_pct)`
- State bins: Compressed / Normal / Expanding by 33/66

### 3.4 MPS (0-100)

- `MPS = 0.4LPI + 0.3RPI + 0.3*VRI`
- State bins: Calm/Mixed/Pressure/Extreme by 0-39/40-69/70-84/85-100

## 4) Confidence Computation (must ship)

### 4.1 Layer confidence

Start at 100

Apply penalties:

- stale daily: -30
- stale weekly: -25
- missing feature: -35 each
- data_limited: -25 if `N < min_samples`
- noisy penalty: -10 if VIX 10D rolling std is >= 95th percentile of 5y

Clamp: 0..100

### 4.2 MPS confidence

- `MPS_conf = weighted average of (LPI_conf, RPI_conf, VRI_conf)` with same weights as MPS

### 4.3 UI badges

- conf >= 80: `Normal`
- 50..79: `Data limited`
- <50: `Partial`
